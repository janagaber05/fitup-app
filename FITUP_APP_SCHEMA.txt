FITUP APP - DETAILED APPLICATION SCHEMA
Version: 1.0
Status: Draft for implementation alignment before AR finalization

============================================================
1) SYSTEM OVERVIEW
============================================================
FitUp is a fitness membership and engagement app with:
- Authenticated user dashboard
- Membership plans and billing status
- Branch/location info and live capacity
- Class/session booking
- Progress tracking
- AR equipment guidance (manual + auto detection)

Primary architecture:
- Frontend: React (web) + Capacitor (iOS/Android shell)
- Optional backend: REST/GraphQL API (Node/Firebase/Supabase style)
- AR detection providers:
  - Teachable Machine (classification fallback)
  - Roboflow Workflow (object detection for auto tracking)
- Media source:
  - Uploaded user videos (local/object storage)
  - Sketchfab embedded 3D training model

============================================================
6) AR RENDERING SCHEMA (LOGICAL PIPELINE)
============================================================

Pipeline A - Manual AR:
1) User opens AR page
2) User toggles manual mode
3) User taps machine 3D button
4) App opens tutorial + hotspots + demo media
5) Optional: open Sketchfab 3D embed overlay

Pipeline B - Auto Detection AR:
1) Camera frame captured
2) Provider inference (Roboflow first, TM fallback)
3) Label normalized via alias map -> canonical machine id
4) Anchor + bounding box derived from detection
5) Smoothing applied
6) Hotspots projected inside bounding box
7) User opens guide/demo

============================================================
7) MEDIA SCHEMA DETAILS
============================================================

Accepted upload:
- video/mp4, video/webm (prefer mp4 h264)
- max size configurable (e.g. 100MB)

Storage strategy:
- Local blob URL (current prototype)
- Production: object storage (S3/R2/Supabase storage)

Sketchfab:
- store embed URL in machine_demo_media.source_url
- policy for fallback if iframe blocked:
  - open in new tab button

============================================================
8) SECURITY / PRIVACY SCHEMA
============================================================

- Never hardcode API keys in client for production
- Move Roboflow calls to backend proxy for key protection
- Add request rate limiting
- Validate uploaded media mime type and size
- Store minimum personal data needed
- Add role-based permissions for calibration updates

============================================================
9) PERFORMANCE SCHEMA
============================================================

- Detection interval: 800ms-1500ms configurable
- Frame image quality: jpeg 0.65-0.8
- Smoothing factor: 0.2-0.35 blend
- Use throttling for AR event logging
- Lazy load heavy AR/3D components
